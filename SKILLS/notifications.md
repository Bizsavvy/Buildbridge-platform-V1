# System Notifications

Notifications are essential for providing closure and reassurance in the funding journey.


## 1. Usage Standards
Use non-blocking, dismissible notifications for:
* **Successful Actions**: Pledge confirmed, Vouch submitted, Proof-of-Use uploaded.
* **Validation Errors**: Clear, non-patronizing feedback on form errors.
* **Trust Milestones**: Notifying a tradesperson when they earn a new "Verified" badge.
* **API Failures**: Providing recovery paths when network connections drop.

## 2. Tone and Timing
* **Dignified Celebration**: Success notifications for funding should feel like a community win: "This need is now 100% funded!".
* **Persistence**: Notifications should remain visible for 5–10 seconds to ensure they are read, but must never block the primary navigation.
* **Visual Distinction**: 
    * **Success**: Uses "Trust" or "Progress" tokens.
    * **Error**: Uses "Alert" or "Review" tokens.
    * **Info**: Uses neutral surface-variant tokens.
