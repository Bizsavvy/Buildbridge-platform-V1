# Form Standards

Forms must be designed for low-literacy users and mobile-first input, prioritizing "Progressive Disclosure".

## 1. Layout and Interaction
* **One Field Per Screen**: For complex flows like "Need Creation," use a single-column layout with one primary question per screen to reduce cognitive load.
* **Visual Selectors**: Use tap-able cards or icons for categories (e.g., choosing "Tailor" or "Mechanic") instead of long dropdowns.
* **Labels and Guidance**: Every field must have a clear label. Use guided prompts for "Short Stories" to help tradespeople describe their need with low writing burden.

## 2. Submission and Errors
* **Loading States**: Submit buttons must transition to a loading state to prevent double-submissions.
* **Inline Validation**: Errors should appear directly below the relevant field in an actionable format.
* **Profile Previews**: Always show a "Profile Preview" or "Listing Preview" before the final submission to give the user a sense of professional pride.
